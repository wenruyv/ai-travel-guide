## 1. 项目概览

### 1.1 一句话定位
**旅伴** 是"攻略即契约 · 搭子即队伍 · 空间即旅程"的小程序。用户在这里：
- **挑攻略** → AI 现编 or 找相似；
- **约搭子** → 拼成同行即自动建空间；
- **共编共记** → 多人共写一份行程、一本旅账；
- **种草转化** → 自然嵌入京东酒旅 / 行前装备 / PLUS 会员。

### 1.2 形态参数
- **平台**：微信小程序（原生 wxml/wxss/js）
- **页面规模**：14 个页面，6 大模块，5 个 tabBar
- **主色**：`#2f80ed`（京东蓝），强调色 `#e1251b`（京东红，京东系转化场景）
- **状态层**：纯前端 + `wx.storageSync` 本地持久化（无后端依赖）
- **数据源**：`miniprogram/mock/data.js` 提供攻略 / 搭子 / 旅账 / 京东订单 / PLUS 会员 mock

### 1.3 顶层心智模型
```
攻略 (契约)  →  搭子 (队伍)  →  空间 (旅程)  →  旅账 (结算)  →  京东 (种草)
   ↑                                                              ↓
   └──────────────── 问真人楼主 / AI 助手 / 找相似攻略 ←────────────┘
```

## 3. 核心模块详解

### 3.1 模块 A：内容发现层（攻略）

#### 入口与召回
- **发现页** ([`discover/discover.wxml`](miniprogram/pages/discover/discover.wxml:1))：顶部 AI 攻略生成 banner + 双列瀑布流，点击攻略卡 → 详情。
- **攻略社区** ([`guide/list/list.wxml`](miniprogram/pages/guide/list/list.wxml:1))：关键词搜索 + 列表，source badge 区分 AI / 小红书 / 马蜂窝 / 原创。
- **攻略详情** ([`guide/detail/detail.wxml`](miniprogram/pages/guide/detail/detail.wxml:1))：13.5 KB，最大单页；包含 AI 行程助手、节点详情、问楼主、京东种草。

#### AI 行程生成（双模式入口）
[`aiGuide/aiGuide.wxml`](miniprogram/pages/aiGuide/aiGuide.wxml:1) 在同一行提供 **"AI 一键生成" + "找相似攻略"** 双按钮：

| 入口 | 触发 | 产出 | 适用心智 |
|---|---|---|---|
| 🤖 AI 一键生成 | `onGenerate` | AI 全新行程（mock 生成器） | 想偷懒 / 条件冷门 |
| 🧩 找相似攻略 | `onFindSimilar` | 既有攻略 Top 4 + 命中点 | 想直接复用 / 找共鸣 |

**找相似攻略打分算法** ([`aiGuide.js#matchSimilarGuides`](miniprogram/pages/aiGuide/aiGuide.js:1))：
| 维度 | 权重 | 命中规则 |
|---|---|---|
| 目的地 | 40 | 精确包含 +18，同省份 +12 |
| tags 命中 | 25 | 每命中 +8，封顶 25 |
| 天数差距 | 15 | ≤1 满分 15，≤2 得 10，≤3 得 6 |
| 预算档 | 10 | 0.6~1.5× 同档，0.3~2.5× 相邻档 |
| 节奏 / 同行人 | 10 | 亲子 / 情侣 / 特种兵 / 深度 命中 |

→ 按 score 降序取 **Top 4**，score=0 过滤；候选卡展示 **匹配度 %** + **最多 3 个命中点 chip**。

#### 攻略详情页核心机制（差异化护城河）
按 [`project_ask_author.md`](.joycode/memory/project_ask_author.md) 记忆，详情页互动核心是 **"问真人楼主"**：
- 每步节点下方有小字问答列表（"问小明：这条路线雨天怎么办？答：改成室内…");
- 头像 + 昵称透明，**不是 AI、不是弹幕**；
- 这是项目不可变更的差异化设计原则。

### 3.2 模块 B：搭子拼成同行

#### 搭子广场 ([`buddy/buddy.wxml`](miniprogram/pages/buddy/buddy.wxml:1))
- 四维筛选：目的地 / 日期 / 兴趣 / 预算
- 卡片信息：头像 / 昵称 / 标签 / 匹配度
- 触发动作：**私聊** → chatDetail / **拼成同行** → 自动建空间

#### 拼成即建空间 ([`buddy.js#_createSpaceOnPair`](miniprogram/pages/buddy/buddy.js:1))
- 拼成瞬间基于选中攻略创建 `travelSpace`
- 双向关联 `conversationSpace[convId] = spaceId`
- 这是"攻略即契约、搭子即队伍、空间即旅程"主链路的实现关键

#### 发布搭子 ([`postBuddy/postBuddy.wxml`](miniprogram/pages/postBuddy/postBuddy.wxml:1))
- 必填攻略（**攻略即契约**）；
- 表单：昵称 / 头像 / 目的地 / 日期 / 预算 / 兴趣 chip / 介绍；
- 提交后写入 `posted_buddies` storage。

### 3.3 模块 C：旅行空间（核心创新）

[`space/space.wxml`](miniprogram/pages/space/space.wxml:1) 三 Tab 共 14.5 KB，是全项目最重的单页：

| Tab | 功能 | 数据结构 |
|---|---|---|
| 🗓 行程 | 按天节点流 + AI 行程助手 + 节点共编留痕 | `space.nodes` + `nodeOverrides` + `spaceEdits` |
| 💰 旅帐 | 多笔记账 + 智能 AA + 京东订单导入 | `bill_entries_v2`（storage） |
| 👥 成员 | 头像 / 角色 / 拼成来源 / 加入时间 | `space.members` |

**核心机制 1：节点共编留痕** ([`space.js`](miniprogram/pages/space/space.js:1))
- 谁编辑了哪个节点、编辑前后的 diff、被谁接受 → 写 `spaceEdits`；
- 不直接覆盖原节点，而是用 `nodeOverrides` 做局部覆盖；
- 保留所有人的协作痕迹（"谁动了我的攻略"）。

**核心机制 2：差额 AA 贪心配对**
```js
// 伪代码：sort debtors desc + creditors desc
while (debtors && creditors) {
  pay = min(|debtor|, creditor);
  push {from: debtor, to: creditor, amount: pay};
  adjust balances; pop done;
}
```
→ **最少笔数净转账**，对比 O(n²) 列举方案，3~6 人组只需 1~2 笔。

**核心机制 3：AI 行程助手面板**
- 按天推荐酒店 / 餐厅 / 装备；
- 与 detail 页 AI 助手共享同一提示词模板；
- 不替代攻略，仅做"今日建议"。

### 3.4 模块 D：消息与旅账

#### 会话层
- [`chat/chat.wxml`](miniprogram/pages/chat/chat.wxml:1) - 会话列表 + **我的旅账 hero card**；
- [`chatDetail/chatDetail.wxml`](miniprogram/pages/chatDetail/chatDetail.wxml:1) - 单聊 + 顶部 **空间卡片 / 建空间 CTA**；
- 两者通过 `chat_conversations` storage 同步，**拼成同行 / 发布搭子** 等场景会自动追加会话与首条消息。

#### 旅账层
- [`bill/bill.wxml`](miniprogram/pages/bill/bill.wxml:1) - 总览 / 明细 / 智能 AA 结算 / 京东订单导入；
- 数据存 `bill_entries_v2`；首次进入从 `mock.bill.entries` 搬入；
- **京东订单导入**：将 `jdOrders`（住宿/门票/装备）按人均分自动入账。

### 3.5 模块 E：京东种草转化

#### 京东酒旅 ([`hotel/hotel.wxml`](miniprogram/pages/hotel/hotel.wxml:1))
- 4 tab：酒店 / 机票 / 门票 / 度假；
- 卡片含 `返京豆` 徽标、`原价划线`、PLUS 标识；
- 转化点 1：酒店预订弹窗"邀请搭子拼房再减 100 元"；
- 转化点 2：机票抢低价监控；
- 转化点 3：门票 / 度假一键加车。

#### 行前装备（攻略详情内嵌）
- 详情页底部"行前清单"：冲锋衣 / 徒步鞋 / 充电宝 / 防晒 → 跳转京东商品（mock）；
- 京东订单可一键导入旅账。

#### PLUS 会员体系 ([`mine/mine.wxml`](miniprogram/pages/mine/mine.wxml:1))
- 等级 / 成长值 / 进度条；
- 签到 +5 京豆 / 任务（发攻略、找搭子、记旅账、评行程）领京豆；
- 会员特权：返豆倍数 / 专属价 / 拼房减额。

### 3.6 模块 F：打卡社区（轻量 UGC）

[`checkin/checkin.wxml`](miniprogram/pages/checkin/checkin.wxml:1)：
- 帖子流 + 足迹地图入口（mock 占位）；
- 多图上传（最多 9 张，emoji 占位）+ 点赞 + 评论 + 标签；
- 私聊作者跳 [`postDetail`](miniprogram/pages/postDetail/postDetail.wxml:1) → 私聊入口。

## 5. 关键算法

### 5.1 找相似攻略（5 维加权）
详见 §3.1。设计要点：
- **目的地权重最高**（40）：相似性主要靠地理决定；
- **tags 辅助语义**（25）：亲子 / 徒步 / 文艺 等抽象需求；
- **天数 / 预算 / 节奏** 细调（15+10+10）：决定执行可行性；
- 命中点 chip **透明化**："匹配度 87% · 目的地命中 · tags 命中"。

### 5.2 攻略召回 3 级瀑布
[`guide/detail/detail.js`](miniprogram/pages/guide/detail/detail.js:1) 在无攻略 ID 时启用：
```
L1 本地攻略 (mock.guides)  →  L2 原始攻略 (raw)  →  L3 AI 临时生成
```
- L1 命中 → 直接渲染；
- L2 命中 → 提示"转载来源"；
- L3 兜底 → 展示"AI 现编"标识 + 允许用户反馈。

### 5.3 差额 AA 贪心
详见 §3.3 核心机制 2。**最少笔数**的数学保证：
- 每人净额 = 垫付 − 应摊；
- 排序后最大正+最大负配对，迭代到 0；
- 复杂度 O(n log n)，n=3~6 时仅 1~2 笔。

### 5.4 京东种草路径转化漏斗
```
攻略详情 (种草点 1: 行前装备)
  → 酒店预订弹窗 (种草点 2: 拼房减额)
    → PLUS 会员卡 (种草点 3: 返豆)
      → 任务领豆 (种草点 4: 任务体系)
        → 长期留存
```

## 7. 设计原则

### 7.1 攻略即契约
- 发搭子必带攻略；
- 拼成同行时攻略成为空间骨架；
- 节点共编在骨架上叠加（不覆盖）。

### 7.2 搭子即队伍
- 拼成自动建空间 + 自动建群；
- 队伍里共享攻略、共编行程、共记旅账。

### 7.3 空间即旅程
- 一个空间 = 一次旅行的全量协作；
- 3 tab 覆盖"前（攻略）— 中（行程/旅帐）— 后（成员/复盘）"。

### 7.4 问真人楼主（非 AI）
详见 [`project_ask_author.md`](.joycode/memory/project_ask_author.md)。**核心差异化护城河**：
- 节点下方小字问答列表，楼主头像 + 昵称透明；
- 不抢 AI 风头，AI 助手单独面板；
- 信任感 > 信息密度。

### 7.5 找相似攻略优先于 AI 生成
- 双按钮并列，主按钮 AI 一键生成（flex 1.4）；
- 次按钮找相似攻略（flex 1）；
- 心智：先挑现成 → 没合适的 → 让 AI 现编。
