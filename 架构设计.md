# 旅伴 - 整体系统架构设计（社区攻略为主体版）

## 0. 产品定位与聚焦策略

> **一句话定位**：把"别人花 10 小时做的攻略"变成"你 10 秒钟能读到的本地指南"——
> 先有海量社区攻略可按行精读，AI 生成只是当你懒得挑时的一键兜底。
>
> **核心心智**：
> ① 进入「攻略」tab → 浏览**真实楼主**写的小红书/马蜂窝笔记（按行展示原始内容）
> ② 点节点「问楼主」→ 真人回复列在该步下方（不是 AI、不是弹幕）
> ③ 当不想翻时 → 切到「发现」页顶部 AI 入口 → 一键生成结构化攻略
> ④ 攻略详情页底部"订同款酒店"+"行前装备加购" → 京东酒旅转化

### 0.1 聚焦原则（5C 评审对齐）

| 评审维度 | 权重（海选）| 我们的策略 |
|---|---|---|
| 创新性 | 50% | **社区攻略按行精读 + 问真人楼主** + AI 攻略生成兜底 + AI 行程助手，三层差异化 |
| 聚焦性 | 30% | tabBar 5 个保持稳定，「攻略」tab 直接进入社区列表；AI 入口仅在发现页顶部 banner |
| 价值贡献 | 20% | 攻略节点中嵌入**京东酒旅/装备推荐位**，读攻略即转化 |

### 0.2 核心交互闭环

```
发现页（discover）
   │  顶部 AI 入口 banner（一键生成攻略，二级入口）
   │  推荐打卡瀑布流
   ↓
攻略 tab（guide/list）  ← 最外层默认入口
   │  社区攻略（xhs/mafengwo）+ 本地原创 + AI 生成（带标签）
   │  列表卡片区分来源
   ↓
攻略详情页（guide/detail）  ← 双模式展示
   │  社区攻略：原始笔记按行展示（主体） + AI 智能解析（折叠辅助）
   │  本地/AI 攻略：行程时间轴（主体）
   │  节点下：问真人楼主 / 就这一步问楼主
   │  底部：京东订同款酒店 + 行前装备加购
   │  AI 攻略才显示：AI 行程助手 FAB
   ↓
找搭子 / 私聊楼主 / 加入旅账
```

## 2. 核心功能模块

### 2.1 🤖 AI 攻略生成引擎（**二级入口 / 兜底能力**）

> AI 生成不是产品主入口，而是当用户「懒得翻」时的一键兜底。发现页顶部 banner 引导进入。

| 功能 | 说明 |
|------|------|
| 一键生成 | 输入目的地/天数/预算/同行人/偏好 → AI 生成完整攻略 |
| 模板库 | 内置 30+ 场景模板（亲子/情侣/独自/摄影/赏樱/滑雪…） |
| 节点级生成 | 每天的时间/交通/住宿/餐饮/景点分段生成 |
| 一键优化 | 生成后可对单一节点追加指令"换成民宿/加景点/避开人多的" |
| 真实数据兜底 | 热门路线直接调用已验证的攻略数据（如 g1 川西/g3 关西）命中后 redirect 到本地攻略 |
| 入口 | 仅在 `pages/discover/discover` 顶部 banner 出现 |

**AI 生成 Prompt 核心**：

```
[系统角色] 你是一位 10 年经验的旅行规划师，熟悉全球目的地。
[输入] 目的地={x}, 天数={n}, 预算={y}, 同行人={z}, 偏好={tags}
[约束]
- 每天 2-4 个节点，type ∈ {location, cost, transport}
- 必须包含 timeSlot / transport / accommodation / meal 四个字段
- 餐饮需具体到餐厅或菜品，不写"当地美食"这种泛词
- 交通需含方式+时长+参考费用
[输出] JSON 结构化攻略
```

### 2.2 攻略社区模块（**主入口 / 产品核心**）

> 攻略 tab 直接进入社区列表：真实楼主写的小红书/马蜂窝笔记为主体，本地原创攻略为支撑，AI 生成攻略仅占少数。

| 功能 | 说明 |
|------|------|
| 社区攻略浏览 | 卡片显示来源标签（📕小红书 / 🧭马蜂窝 / 🤖AI生成 / ✍️原创）|
| 关键词检索 | 按目的地/玩法/标签搜索 |
| **双模式详情页** | 见下表，**这是本版本最大变化** |
| 问真人楼主 | **核心互动**，问答小字列在每步下方（不是 AI、不是弹幕）|
| 复制修改 | 一键复制他人攻略，生成个人版本可编辑 |
| AI 行程助手 | **仅 AI 攻略详情页**显示浮动 FAB，社区攻略时不出现 |

#### 2.2.1 详情页双模式展示

| 攻略来源 | 主体内容 | 辅助内容 | AI 助手 FAB |
|----------|----------|----------|-------------|
| **xhs / mafengwo（社区）** | 原始笔记按行展示（DayN 行高亮+左边框） | AI 智能解析结果（默认折叠，点"展开 AI 智能解析"按钮显示） | ❌ 不显示 |
| **local（本地原创）** | 行程时间轴（节点/时间/交通/住宿/餐饮） | — | ❌ 不显示 |
| **ai（AI 生成）** | 行程时间轴（节点/时间/交通/住宿/餐饮） | — | ✅ 突出显示 |

**社区攻略按行展示实现**：
- `detail.js` 在 `onLoad` 中将 `guide.rawContent` 按 `\n` 拆分为 `rawLines` 数组
- 每行打标：`isDay`（`/^[Dd]ay\s*\d+/`）、`isEmpty`
- `detail.wxml` 通过 `wx:for="{{rawLines}}"` 渲染，`raw-line-day` 样式高亮 DayN 标题
- 解析结果通过 `parseRawGuide(guide)` 实时计算（g6-g9 raw 笔记入详情页时自动解析），AI 解析节点放在 `parsed-section` 默认折叠

## 4. API 设计原则

- RESTful 风格，JSON 数据格式
- JWT Token 认证（小程序登录后换取）
- 统一响应格式：`{code, message, data}`
- 分页：Page + Size 参数，返回 `{list, total, has_more}`
- 文件上传：独立上传接口，返回 URL 后关联业务数据
- **AI 接口特殊规范**：`POST /api/ai/guide/generate` 走 SSE 流式响应，支持中断重连

## 5. 技术选型理由

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | 微信小程序 | 零安装门槛，社交传播强，原生能力全 |
| 后端 | FastAPI | Python 生态成熟，异步高性能，AI 集成方便 |
| 数据库 | PostgreSQL | 关系型数据为主，JSONB 支持灵活字段 |
| 缓存 | Redis | 会话、排行榜、消息队列、热点数据 |
| AI | Claude API | 推理能力强，适合复杂攻略生成和对话 |
| 向量 | pgvector / Redis | 攻略相似度匹配、语义检索 |
| 文件 | 腾讯云COS | 小程序生态原生支持，CDN 加速 |
| 部署 | 京东DevCloud / 腾讯云 | 根据赛事要求灵活选择 |

## 7. 京东商业化闭环（黑客马拉松核心亮点）

> 目标：将「旅伴」从社交工具升级为**京东酒旅业务的流量入口与转化载体**。
> 商业路径以"读社区攻略"为主要场景，AI 生成作为兜底与促活。

### 7.1 社区攻略驱动的五环闭环

```
社区攻略（xhs/mafengwo，按行精读）  →  攻略详情种草  →  京东酒旅/装备转化  →  旅账关联订单  →  会员/任务促活
   ↑                                          │
   └──────────────── 回流拉新/复购 ←──────────────────────────┘
                                          AI 攻略（兜底）
```

| 环节 | 载体页面 | 商业价值 |
|------|----------|----------|
| ① 社区种草 | `pages/discover/discover` | 顶部 AI 入口 banner（次级） + 推荐打卡 |
| ② 攻略种草 | `pages/guide/list` + `pages/guide/detail` | 列表来源标签引导 + 详情底部"订同款" |
| ③ 酒旅转化 | `pages/hotel/hotel` | 酒店/机票/门票/度假，PLUS 返京豆 |
| ④ 旅账关联 | `pages/bill/bill` | AI 自动分类 + 京东订单导入 |
| ⑤ 会员成长 | `pages/mine/mine` | PLUS 成长值、京豆余额、签到任务 |

### 7.2 关键玩法

- **社区攻略种草位**：攻略详情 `tags` 映射 `gearByTag` 装备商品；`destination` 与酒店 `city` 交叉匹配"订同款"。
- **AI 行程助手加单**（仅 AI 攻略可见）：用户问"我膝盖不好，稻城能换轻量路线吗" → AI 推荐 1-2 个备选酒店 → 一键跳京东酒旅。
- **搭子拼房立减**：结合搭子社交，酒店预订邀请搭子拼房再减 100 元，社交裂变驱动转化。
- **PLUS 返京豆**：订酒店最高返 8%，京豆可抵现，强化 PLUS 会员心智与复购。
- **旅账即订单**：京东酒旅订单一键进旅账，把"记账"变成"消费入口"。
- **任务/签到激励**：每日打开、浏览攻略、收藏酒店、完成预订均发京豆，直接提升 DAU。
